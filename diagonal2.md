# diagonal2: 決定論的衝突回避ルーティング

## 目的

全ポートを orthogonal (直交折れ線) でルーティングし、
**重なりが原理的に発生しない**アルゴリズムを設計する。
交差 (H+V) は許容する。

## 既存仕様の維持

- サブブロック、サブターミナル、サブポートの概念
- サブターミナルの配置と分岐構造
- 交差 OK、同方向重なり NG

## 現行アルゴリズムの問題

1. ポートを順に配置し、衝突時に行/列挿入 → 後発ポートが詰まる
2. 3 セグメント (HVH/VHV) では解けないケースがある
3. グリッドサイズが不確定（expansion で膨張）
4. 停止しないケースがある

## 新アルゴリズム: 専用チャネル割り当て

### 核心アイデア

各ポートの中間セグメント（L-bend の折れ点、Z/U-bend の縦/横移動）に
**専用の行または列**を割り当てる。
同じ行/列を共有するポートがないため、同方向重なりが原理的に発生しない。

### グリッドサイズの決定

ポート一覧から事前にグリッドサイズを確定する。

**行の構成:**
- 行 0: N 境界 (N サブターミナル)
- 行 1..pW: W チャネル (各 W ポート端に 1 行)
- 行 pW+1..pW+pE: E チャネル (各 E ポート端に 1 行)
- 行 pW+pE+1..pW+pE+mH: H 中間チャネル (水平中間セグメント用、各ポート最大 1 行)
- 行 pW+pE+mH+1: S 境界 (S サブターミナル)

**列の構成:**
- 列 0: W 境界 (W サブターミナル)
- 列 1..pN: N チャネル
- 列 pN+1..pN+pS: S チャネル
- 列 pN+pS+1..pN+pS+mV: V 中間チャネル (垂直中間セグメント用、各ポート最大 1 列)
- 列 pN+pS+mV+1: E 境界 (E サブターミナル)

**mH, mV の計算:**
- 各ポートのルートタイプを分析
- 隣接辺ポート (L-bend): 中間セグメント不要 (折れ点は既存チャネルの交点)
- 対辺/同辺ポート (Z/U-bend):
  - isLR(src) → V 中間列が 1 列必要 → mV += 1
  - !isLR(src) → H 中間行が 1 行必要 → mH += 1
- 同辺かつ境界上のポート (W-W, E-E, N-N, S-S):
  - isLR → V 中間列が 1 列必要 → mV += 1
  - !isLR → H 中間行が 1 行必要 → mH += 1

### ルーティング規則

各ポートのルートは以下の形式。全セグメントは専用領域を使う。

#### 1. 隣接辺 (Adjacent): L-bend

例: W[i] → N[j]
- H: sg の行をそのまま使い、dg の列まで水平移動
- V: dg の列をそのまま使い、dg の行まで垂直移動
- 中間セグメント不要。H は sg 専用行、V は dg 専用列。

**衝突が起きない理由:**
- H セグメントは sg の行 (W チャネル行) にある。各 W ポート端は異なる行なので H 同士は重ならない。
- V セグメントは dg の列 (N チャネル列) にある。各 N ポート端は異なる列なので V 同士は重ならない。
- 他ポートの H が dg の列を横切る場合、それは交差 (H+V) であり OK。

#### 2. 対辺 (Opposite) / 同辺 (Same): Z/U-bend

例: W[i] → E[j] (対辺、isLR)
- H1: sg の行で、sg の列から中間列 mc まで水平移動
- V: 中間列 mc で、sg の行から dg の行まで垂直移動
- H2: dg の行で、中間列 mc から dg の列まで水平移動
- mc はこのポート専用の V 中間列。

例: N[i] → S[j] (対辺、!isLR)
- V1: sg の列で、sg の行から中間行 mr まで垂直移動
- H: 中間行 mr で、sg の列から dg の列まで水平移動
- V2: dg の列で、中間行 mr から dg の行まで垂直移動
- mr はこのポート専用の H 中間行。

**衝突が起きない理由:**
- H1/H2 は sg/dg の専用行。他ポートの H はその行を使わない。
- V は専用中間列。他ポートの V はその列を使わない。
- 交差はあり得るが、それは H+V なので OK。

#### 3. 同辺かつ境界 (Same-side border)

例: W[i] → W[j]
- 両方 col 0 (W 境界)。直接 V で繋ぐと境界列で重なる。
- 代わりに Z-bend: H1 → V (専用中間列) → H2 で迂回。
- H1 は sg の専用行、H2 は dg の専用行、V は専用中間列。

### 衝突回避の証明

各セグメントが使う行/列を整理:

| セグメント | 使う行/列 | 専用性 |
|-----------|----------|--------|
| H (sg側) | sg のチャネル行 | 各ポート端で一意 |
| H (dg側) | dg のチャネル行 | 各ポート端で一意 |
| V (中間) | ポート専用の中間列 | ポートごとに一意 |
| H (中間) | ポート専用の中間行 | ポートごとに一意 |
| V (sg/dg列) | sg/dg のチャネル列 | 各ポート端で一意 |

**同方向衝突が起きるのは、2 つのセグメントが同じ行 (H) または同じ列 (V) を共有する場合のみ。**

- 同じ行を共有する H セグメント: sg/dg のチャネル行はポート端ごとに一意。中間行もポートごとに一意。
  → ただし、2 つの異なるポートが同じターミナルを端点として共有する場合
  (例: W0-N0 と W0-E1 で W0 が共通)、sg のチャネル行が同じになりうる。
  これはサブターミナルの分岐で処理: 同じターミナルの複数ポートは異なるチャネル行を持つ。
- 同じ列を共有する V セグメント: 同様に一意。

**結論: 全ポート端のチャネルが一意 (既存のチャネル割り当てで保証) かつ
中間セグメントが専用行/列を持つため、同方向重なりは発生しない。**

### L-bend の衝突可能性の詳細分析

L-bend は中間セグメントを使わないが、2 つの L-bend が衝突する可能性:

Port A: W[a] → N[b], L-bend at (row_a, col_b)
Port B: W[c] → N[d], L-bend at (row_c, col_d)

- Port A の H: row_a, cols 0..col_b
- Port B の H: row_c, cols 0..col_d
- row_a ≠ row_c (異なる W チャネル) → H 同士は別行 → 重ならない ✓

- Port A の V: col_b, rows 0..row_a
- Port B の V: col_d, rows 0..row_c
- col_b ≠ col_d (異なる N チャネル) → V 同士は別列 → 重ならない ✓

Port C: W[a] → S[e] (Wチャネルを共有)
- Port A の H: row_a, cols 0..col_b
- Port C の H: row_a, cols 0..col_e ← 同じ行!

**問題: 同じターミナル (W[a]) から出る 2 つの L-bend ポートが同じ行で H 重なりする。**

**解決: 同じターミナルからの複数ポートは異なるチャネル (異なる行) を割り当てるので、
Port A は row_a1, Port C は row_a2 (a1 ≠ a2) となり重ならない。** ✓

### L-bend と Z-bend の衝突可能性

Port A: W[a] → N[b] (L-bend)
- H: row_a, cols 0..col_b
- V: col_b, rows 0..row_a

Port D: E[c] → W[d] (Z-bend, isLR)
- H1: row_c (E チャネル行), cols from E border to mc
- V: mc (専用中間列), rows row_c..row_d
- H2: row_d, cols mc..0

Port A の H (row_a) と Port D の H1 (row_c) or H2 (row_d):
- row_a は W チャネル行。row_c は E チャネル行。row_d は W チャネル行。
- W と E は別行なので row_a ≠ row_c ✓
- row_a vs row_d: 両方 W チャネル行だが、a ≠ d なら異なる行 ✓
- a = d の場合: W[a] が Port A の sg かつ Port D の dg。
  同じターミナルだが Port A では src、Port D では dst。
  チャネル割り当て: Port A の sg は row_a1、Port D の dg は row_a2 (異なるチャネル)。
  → row_a1 ≠ row_a2 ✓

Port A の V (col_b) と Port D の V (mc):
- col_b は N チャネル列。mc は専用中間列。これらは別領域 → col_b ≠ mc ✓

**結論: L-bend と Z-bend の間でも衝突しない。** ✓

## 実装

### ステップ

1. ポート分析: 各ポートのタイプ (adjacent/opposite/same) を判定
2. チャネル割り当て: 既存の sideCh ベースの連番割り当て (変更なし)
3. 中間チャネル割り当て:
   - Z/U-bend の isLR ポート → V 中間列を連番割り当て
   - Z/U-bend の !isLR ポート → H 中間行を連番割り当て
4. グリッドサイズ確定:
   - nR = 1 (N境界) + pW + pE + mH + 1 (S境界)
   - nC = 1 (W境界) + pN + pS + mV + 1 (E境界)
5. ルーティング: 各ポートの型に応じて placeH/placeV (衝突チェック不要)
6. スパイン描画: 既存ロジック
7. termPos 構築: 既存ロジック + デフォルト埋め

### L-bend の折れ点の決定

L-bend Port: W[a] → N[b]
- sg: (row_a, col_W=0)
- dg: (row_N=0, col_b)
- 折れ点: (row_a, col_b) — sg の行と dg の列の交点

L-bend Port: N[a] → E[b]
- sg: (row_N=0, col_a)
- dg: (row_b, col_E=nC-1)
- 折れ点: (row_b, col_a) — dg の行と sg の列の交点

一般化: isLR(src) なら折れ点 = (sg.r, dg.c)。!isLR(src) なら折れ点 = (dg.r, sg.c)。
(これは現行と同じ)

### expansion 不要

グリッドサイズが事前に確定し、各セグメントが専用領域を使うため、
ルーティング中の行/列挿入は不要。while ループも不要。
**アルゴリズムは O(nports) で必ず停止する。**
