# 目的
斜め線で描かれている port を全て diagonal な折れ線で表現する。

# 方法
nxport, nyport は曲線のまま。

ブロックの W, E, S, N のターミナルに繋がっているポート数を p_W, p_E, p_S, p_N とする。

p_SN=max(p_S,p_N), p_WE=max(p_W,p_E)とする。
ブロックを (p_SN+2)x(p_WE+2) のサブブロックに分ける。

Wiポートはブロック内アドレス(     0,i+1) のサブブロックに接続される。
Eiポートはブロック内アドレス(p_WE-1,i+1) のサブブロックに接続される。
Siポートはブロック内アドレス(i+1,     0) のサブブロックに接続される。
Niポートはブロック内アドレス(i+1,p_SN-1) のサブブロックに接続される。

# サブターミナル
一番外側にあるサブブロックは、ひとつの terminal から複数の port が出ていた場合の分岐用のスペースである。
Wiターミナルにk個のポートが繋がっていた時には、(0,i)から(0,i+k-1)に

┌ i+k-1:L字型 (下と右を結合)
├ i+k-2:分岐型 (下と上と右を結合)
├ i+k-3:分岐型 (下と上と右を結合)
:
├ i+  2:分岐型 (下と上と右を結合)
├ i+  1:分岐型 (下と上と右を結合)
┴ i+  0:分岐型 (左と右と上を結合)

という分岐路を形成する。これにて出来上がったサブブロック(0,i)--(0,i+k-1)のことをサブターミナルと呼ぶことにする。
i
# サブポート
WからNやSに接続するポートは、サブターミナルから接続先のサブターミナルのx座標の位置まで直進し、
そこから直角に曲がり、直進して接続先のサブターミナルに入る。
このサブブロックに存在するセグメントをサブポートと呼ぶことにする。

WからEに接続するポートは、サブターミナルから左右の中央位置までまっすぐ直進し、
直角に曲がり、そこから接続先のサブターミナルの高さまで上下移動をして、
またまっすぐ接続先のサブターミナルに入る。


# 重複の回避
基本的には白紙のサブブロックから初めてポートを１つ１つ
サブポートとして書いていくことでサブポートの結線図が出来る。
しかし、複数のサブポートが重複していると、接続が分かりにくくなるので、その場合は、下記のように行や列を挿入して回避する。

## 手順
サブターミナルから出て、１サブブロックずつサブブロックを伸ばしていくことで、サブポートの章に書いていた手順で書いていく。
### 交差
縦方向のサブポートと横方向のサブポートが重なった場合は、交差すればいいので、┼のかたちで描画するので問題ない。
### 交差以外
それ以外のサブポート同士や、それ以外のサブポートと交差ポートが衝突した場合は下記の方法で避ける。

次に上への直角曲がりを予期しながら右に直進しているときに衝突したら、上に１行新しい行を追加して、そちらに乗り換えて右に進行を続ける。
次に下への直角曲がりを予期しながら右に直進しているときに衝突したら、下に１行新しい行を追加して、そちらに乗り換えて右に進行を続ける。
