# ミンスキーダブリングマシン

繰り返し迷路上に2カウンタ ミンスキーレジスタマシンを符号化する。各ダブリングサイクルは x レジスタを一時記憶として `y ↦ 2y + 1` を計算する。k サイクル後のパス長は O(2^k) で成長し、nterm = 12k + 3 であるため nterm に対して指数的。

x の動き（W→E / E→W ポート）でレジスタ x、y の動き（S→N / N→S ポート）でレジスタ y を表現し、境界ブロック（x=0 の nx、y=0 の ny）でゼロテスト分岐を実現する。

## 具体例

- [k=5](5-ja.md) — 5回ダブリング、nterm=61、パス長 662
