# Counter Pump nterm=6

nterm=6, w=5, path length=257

## Mechanism

The counter pump uses asymmetric accumulation and release of the
y-coordinate as a counter. With nterm = n = 6 terminals per side
and return width w = 5:

- **Forward cycle**: Slowly accumulates y while moving east.
  Each cycle: (+1, +4) displacement in 8 steps.
- **Descent**: Rapidly consumes y via N0->S0 (1 step per y).
- **Climb**: Re-enters the S-N chain to reach N1 for return.
- **Return**: Moves west in sub-cycles of w=5 hops, losing 1 in y each time.

### Haskell

```haskell
-- Counter pump nterm=6, w=5
-- Registers: x (horizontal), y (vertical)
-- Initial call: cpump6 (0, 1, 0)
-- Total steps: 257

cpump6 :: (Int, Int, Int) -> (Int, Int, Int)
--        x    y    pc       x    y    pc

-- Forward cycle: x+=1, y+=4 per cycle (8 steps each)
cpump6 (  x,   y,   0) = cpump6 (  x,   y,   1)     -- W0->N0: enter block
cpump6 (  x,   y,   1) = cpump6 (  x, y+3,   2)     -- N0->N4, climb S4->N3->N2->N1
cpump6 (  x,   y,   2) = cpump6 (x-1,   y,   3)     -- S1->E5, E5->W4: go west
cpump6 (  x,   y,   3) = cpump6 (x+1,   y,   0)     -- W4->E0: bounce east

-- Descent: y -= 1 per step (N0->S0)
cpump6 (  x,   y,   4) = cpump6 (  x, y-1,   4)     -- N0->S0: descend
-- (transition from forward to descent is forced by BFS optimality)

-- Climb after descent: N0->N4, S4->N3, S3->N2, S2->N1
cpump6 (  x,   y,   5) = cpump6 (  x, y+3,   6)     -- climb to N1

-- Return sub-cycle: x -= 5, y -= 1 (6 steps each)
-- N1->W0, E0->W5, E5->W4, E4->W3, E3->W2, then E2->S1
cpump6 (  x,   y,   6) = cpump6 (x-5, y-1,   6)     -- non-final return
-- Final return: N1->W0, E0->W5, E5->W4, E4->W3, E3->W2, W2->W1
cpump6 (  x,   y,   6) =       (x-5,   y,  -1)      -- HALT at goal (0,1)
```

Note: The Haskell above is a simplified model. The actual maze uses a single
program counter (terminal index) and the BFS solver determines the optimal
phase transitions. Forward and descent both use the N0 terminal, with the
BFS choosing N0->N4 (forward/climb) or N0->S0 (descent) based on shortest path.

### Port Structure

```
Forward cycle (8 steps per cycle):
  W0 -> N0         enter block from west
  N0 -> N4         jump to high index
  S4 -> N3         y+1 (boundary cross)
  S3 -> N2         y+1
  S2 -> N1         y+1
  S1 -> E5         switch to east side
  E5 -> W4         go west (x-1)
  W4 -> E0         bounce east (x+1)

Descent:
  N0 -> S0         y-1 (repeated)

Return chain (w=5 westward hops):
  N1 -> W0         x-1 (hop 1)
  E0 -> W5         x-1 (hop 2)
  E5 -> W4         x-1 (hop 3, shared with forward)
  E4 -> W3         x-1 (hop 4)
  E3 -> W2         x-1 (hop 5)

End conditions:
  E2 -> S1         non-final: y-1, loop back to N1
  W2 -> W1         final: exit to goal E1@(0,1)
```

### Growth

With n = nterm:

```
w = 3 (fixed)   : T = 3(2n^2 - 3n - 2) - 3 = 6n^2 - 9n - 9     = Theta(n^2)
w = n-1 (max)   : T = (n-1)(2n^2 - 3n - 2) - 3 = 2n^3 - 5n^2 + n - 1 = Theta(n^3)
```

For n=6: T = 257 = 2(216) - 5(36) + 6 - 1 = 432 - 180 + 6 - 1 = 257. Verified.
