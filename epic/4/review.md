# Epic 4: レビュー結果

## 結果: PASS (v1.5)

## 試行した方法と結果

| バージョン | 方法 | 結果 |
|-----------|------|------|
| v1.1 | termPos 位置合わせ (ポート参加時のみ) | Bug: 36.6% 位置ずれ |
| v1.2 | termPos 位置合わせ (デフォルト後) + ブリッジスパイン | Bug: 88.5% スパイン重なり |
| v1.3 | 全 Z-bend + chG 位置合わせ | Bug: 77.3% ルート重なり |
| v1.4 | L/Z ハイブリッド + chG 位置合わせ | Bug: 71.8% ルート重なり |
| **v1.5** | **v1.0 ルーティング + termPos のみ位置合わせ** | **1000/1000 PASS** |

## 学んだこと

- chG (ルーティンググリッド座標) の位置合わせは diagonal2 の衝突回避証明を破壊する
- ブリッジスパインは境界線上に集中し重なる
- **ルーティングは一切変更せず、表示レイヤ (termPos) のみで位置合わせするのが正解**

## v1.5 の変更内容

1. routeBlockPorts: v1.0 と同一 (L-bend + Z-bend, 別行/別列)
2. termPos: E[i].y = W[i].y, S[i].x = N[i].x (表示のみ)
3. chG 位置合わせ: なし
4. ブリッジスパイン: なし

## テスト結果

- ランダム 1000: 0 overlap (routes + spines 全チェック)
- termPos 位置ずれ: 0
